Урок 8: Обработка ошибок

Учим программы правильно реагировать на проблемы

Зачем обрабатывать ошибки?

Ошибки (исключения) возникают, когда программа сталкивается с неожиданной ситуацией. Без обработки ошибок программа аварийно завершается.

Программа работает нормально
↓
Возникает ошибка
↓
Ошибка перехвачена и обработана
↓
Программа продолжает работу 
# Без обработки ошибок - программа упадет
число = input("Введите число: ")
результат = 10 / int(число)  # Ошибка, если ввести 0 или текст
print(f"Результат: {результат}")

# С обработкой ошибок - программа продолжит работу
try:
    число = input("Введите число: ")
    результат = 10 / int(число)
    print(f"Результат: {результат}")
except:
    print("Что-то пошло не так, но программа не упала!")

Конструкция try-except

Основной способ обработки ошибок в Python:

# Общая форма
try:
    # Код, который может вызвать ошибку
    рискованный_код()
except:
    # Код, который выполнится при ошибке
    обработка_ошибки()

# Конкретный пример
try:
    число = int(input("Введите число: "))
    print(f"Вы ввели: {число}")
except ValueError:
    print("Это не число! Пожалуйста, введите целое число.")
except ZeroDivisionError:
    print("На ноль делить нельзя!")
except Exception as e:
    print(f"Произошла ошибка: {e}")

Типы исключений

Python имеет множество встроенных типов исключений:

# SyntaxError - ошибка синтаксиса
# print("Привет)  # Ошибка: не закрыта кавычка

# NameError - имя не найдено
try:
    print(несуществующая_переменная)
except NameError:
    print("Переменная не существует")

# TypeError - неверный тип
try:
    результат = "текст" + 5
except TypeError:
    print("Нельзя складывать строку с числом")

# ValueError - неверное значение
try:
    число = int("не число")
except ValueError:
    print("Невозможно преобразовать в число")

# ZeroDivisionError - деление на ноль
try:
    результат = 10 / 0
except ZeroDivisionError:
    print("Деление на ноль невозможно")

# IndexError - индекс вне диапазона
try:
    список = [1, 2, 3]
    элемент = список[10]
except IndexError:
    print("Индекс выходит за пределы списка")

# KeyError - ключ не найден в словаре
try:
    словарь = {"ключ": "значение"}
    значение = словарь["несуществующий_ключ"]
except KeyError:
    print("Ключ не найден в словаре")

# FileNotFoundError - файл не найден
try:
    with open("несуществующий_файл.txt", "r") as файл:
        содержимое = файл.read()
except FileNotFoundError:
    print("Файл не найден")

Полная форма: try-except-else-finally

Расширенная конструкция для полного контроля:

try: Код, который может вызвать ошибку
except: Обработка ошибки
else: Выполняется если НЕ было ошибки
finally: Выполняется ВСЕГДА 
def безопасное_деление(делимое, делитель):
    try:
        результат = делимое / делитель
    except ZeroDivisionError:
        print("Ошибка: деление на ноль!")
        return None
    except TypeError:
        print("Ошибка: неверный тип данных!")
        return None
    else:
        print("Деление выполнено успешно!")
        return результат
    finally:
        print("Этот код выполнится в любом случае")

# Примеры использования
print(безопасное_деление(10, 2))    # Успешно
print(безопасное_деление(10, 0))    # Ошибка деления на ноль
print(безопасное_деление(10, "текст"))  # Ошибка типа

# Практический пример с файлами
def читать_файл(имя_файла):
    try:
        файл = open(имя_файла, "r", encoding="utf-8")
    except FileNotFoundError:
        print(f"Файл {имя_файла} не найден")
        return None
    except PermissionError:
        print(f"Нет доступа к файлу {имя_файла}")
        return None
    else:
        содержимое = файл.read()
        print(f"Файл {имя_файла} успешно прочитан")
        return содержимое
    finally:
        if 'файл' in locals() and not файл.closed:
            файл.close()
            print("Файл закрыт")

Создание собственных исключений

Вы можете создавать свои типы ошибок:

# Создание простого исключения
class ОшибкаВозраста(Exception):
    pass

class ОтрицательныйВозраст(ОшибкаВозраста):
    """Исключение для отрицательного возраста"""
    pass

class СлишкомБольшойВозраст(ОшибкаВозраста):
    """Исключение для слишком большого возраста"""
    pass

# Использование
def установить_возраст(возраст):
    if возраст < 0:
        raise ОтрицательныйВозраст("Возраст не может быть отрицательным")
    if возраст > 150:
        raise СлишкомБольшойВозраст("Возраст слишком большой")
    print(f"Возраст установлен: {возраст}")

# Тестирование
try:
    установить_возраст(-5)
except ОтрицательныйВозраст as e:
    print(f"Ошибка: {e}")

try:
    установить_возраст(200)
except СлишкомБольшойВозраст as e:
    print(f"Ошибка: {e}")

# Более сложное исключение
class ОшибкаБанковскогоСчета(Exception):
    def __init__(self, баланс, сумма):
        self.баланс = баланс
        self.сумма = сумма
        self.сообщение = f"Недостаточно средств: баланс {баланс}, запрошено {сумма}"
        super().__init__(self.сообщение)

class БанковскийСчет:
    def __init__(self, начальный_баланс):
        self.баланс = начальный_баланс
    
    def снять(self, сумма):
        if сумма > self.баланс:
            raise ОшибкаБанковскогоСчета(self.баланс, сумма)
        self.баланс -= сумма
        print(f"Снято {сумма}. Новый баланс: {self.баланс}")

# Использование
счет = БанковскийСчет(1000)
try:
    счет.снять(1500)
except ОшибкаБанковскогоСчета as e:
    print(e.сообщение)
    print(f"Не хватает: {e.сумма - e.баланс}")

Контекстные менеджеры (with)

Упрощают работу с ресурсами и автоматически обрабатывают ошибки:

# Работа с файлами без with (плохой способ)
try:
    файл = open("пример.txt", "w")
    файл.write("Привет, мир!")
finally:
    if 'файл' in locals():
        файл.close()

# Работа с файлами с with (хороший способ)
with open("пример.txt", "w") as файл:
    файл.write("Привет, мир!")
# Файл автоматически закроется, даже если произойдет ошибка

# Создание собственного контекстного менеджера
class Таймер:
    def __init__(self, название):
        self.название = название
    
    def __enter__(self):
        import time
        self.начало = time.time()
        print(f"Начало: {self.название}")
        return self
    
    def __exit__(self, exc_type, exc_val, exc_tb):
        import time
        self.конец = time.time()
        print(f"Конец: {self.название}")
        print(f"Прошло времени: {self.конец - self.начало:.2f} секунд")
        # Если возвращаем True, исключение будет подавлено
        return False

# Использование
with Таймер("Мой код") as таймер:
    # Здесь выполняется код, который нужно измерить
    сумма = 0
    for i in range(1000000):
        сумма += i
    print(f"Сумма: {сумма}")

# Контекстный менеджер для временного изменения настроек
import sys
from contextlib import contextmanager

@contextmanager
def перенаправить_вывод(новый_файл):
    старый_stdout = sys.stdout
    sys.stdout = новый_файл
    try:
        yield
    finally:
        sys.stdout = старый_stdout

# Использование декоратора
with open("вывод.txt", "w") as файл:
    with перенаправить_вывод(файл):
        print("Этот текст запишется в файл")
        print("А не на экран")

Логирование ошибок

Вместо простого вывода ошибок на экран лучше использовать логирование:

import logging

# Настройка логирования
logging.basicConfig(
    level=logging.DEBUG,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
    handlers=[
        logging.FileHandler('ошибки.log', encoding='utf-8'),
        logging.StreamHandler()  # Также выводить на экран
    ]
)

logger = logging.getLogger(__name__)

def безопасная_функция(данные):
    try:
        if not данные:
            raise ValueError("Пустые данные")
        
        результат = int(данные) * 2
        logger.info(f"Функция выполнена успешно: {результат}")
        return результат
        
    except ValueError as e:
        logger.error(f"Ошибка значения: {e}", exc_info=True)
        return None
    except Exception as e:
        logger.critical(f"Критическая ошибка: {e}", exc_info=True)
        return None

# Разные уровни логирования
logger.debug("Отладочное сообщение")      # Для разработки
logger.info("Информационное сообщение")   # Обычная информация
logger.warning("Предупреждение")          # Что-то необычное
logger.error("Ошибка")                    # Серьезная ошибка
logger.critical("Критическая ошибка")     # Очень серьезная ошибка

# Пример использования
for данные in ["10", "", "текст", "5"]:
    результат = безопасная_функция(данные)
    print(f"Результат для '{данные}': {результат}")

Практическое задание: "Надежный калькулятор"

Усовершенствуйте калькулятор из прошлого урока, добавив обработку ошибок.

Требования:

  1. Добавьте обработку всех возможных ошибок:
    • Деление на ноль
    • Неверный ввод (не числа)
    • Ошибки при работе с файлами
    • Выход за пределы меню
  2. Создайте собственные исключения:
    • ОшибкаКалькулятора
    • ОшибкаВвода
    • ОшибкаФайла
  3. Реализуйте систему логирования:
    • Записывайте все операции в лог-файл
    • Разные уровни логирования для разных событий
  4. Добавьте возможность восстановления:
    • При ошибке предлагать повторить ввод
    • Автоматическое сохранение истории при сбое
  5. Используйте контекстные менеджеры для работы с файлами

Пример структуры:

class ОшибкаКалькулятора(Exception):
    """Базовое исключение для калькулятора"""
    pass

class ОшибкаВвода(ОшибкаКалькулятора):
    """Ошибка ввода данных"""
    pass

class ОшибкаДеленияНаНоль(ОшибкаКалькулятора):
    """Попытка деления на ноль"""
    pass

def безопасное_деление(a, b):
    try:
        if b == 0:
            raise ОшибкаДеленияНаНоль(f"Деление {a} на {b}")
        return a / b
    except ОшибкаДеленияНаНоль as e:
        logger.error(f"Ошибка деления: {e}")
        return None

def получить_число(подсказка):
    while True:
        try:
            ввод = input(подсказка)
            число = float(ввод)
            return число
        except ValueError:
            print("Ошибка! Введите корректное число.")
            logger.warning(f"Неверный ввод: {ввод}")
        except KeyboardInterrupt:
            print("\nВвод прерван")
            raise

Подсказки:

  • Используйте try-except внутри циклов для повторения операций
  • Создайте функцию-обертку для безопасного выполнения операций
  • Используйте модуль logging для записи ошибок в файл
  • Добавьте блок finally для гарантированного сохранения истории
  • Используйте with для работы с файлами логов

Полезные советы

  • Ловите конкретные исключения, а не все через except:
  • Не подавляйте исключения без веской причины
  • Используйте finally для освобождения ресурсов
  • Логируйте ошибки с полной информацией (exc_info=True)
  • Создавайте понятные сообщения об ошибках
  • Используйте контекстные менеджеры для работы с файлами
  • Тестируйте обработку ошибок так же, как основной функционал
