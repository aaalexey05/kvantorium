# Issue Log: 2026-03-04

## Scope
Stabilization and UI cleanup without changing business logic.

## Findings and fixes

### 1) Repeated 404 for Chrome devtools probe
- Problem: Requests to `/.well-known/appspecific/com.chrome.devtools.json` produced 404 and polluted logs.
- Fix: Added dedicated endpoint returning JSON.
- Files:
  - `src/apps/core/views.py`
  - `src/apps/core/urls.py`

### 2) Decorative dinosaurs in header/pages
- Problem: Dinosaur decorations and animations were present in header and hero area.
- Fix: Removed dinosaur markup from header and removed all dinosaur CSS/animations.
- Replacement: Added neutral abstract gradient decoration that does not interfere with layout.
- Files:
  - `src/templates/includes/navbar.html`
  - `src/static/css/main.css`

### 3) Theme refinement for dark/light modes
- Problem: Palette needed more academic and readable look for school/student LMS.
- Fix: Updated centralized theme tokens (dark and light), kept component sizes/radii/shadows structure, improved contrast consistency.
- Files:
  - `src/static/css/main.css`

### 4) Form presentation
- Problem: Input controls looked too plain.
- Fix: Improved visual style of form containers, labels, input/select/textarea hover/focus states, spacing and readability.
- Files:
  - `src/static/css/main.css`

### 5) Course slug stability and 500 on lesson pages
- Problem: Courses with fully Cyrillic titles could get empty slug (`""`) via `slugify`, which broke links and caused `NoReverseMatch` in lesson templates.
- Fix:
  - Added fallback slug generation (`course`, `course-2`, ...) in `Course.save()`.
  - Re-seeded data so existing records receive valid slugs.
- Files:
  - `src/apps/courses/models.py`
  - `src/apps/core/management/commands/seed_demo.py`

### 6) Enrollment/progress actions and demo content
- Problem: Needed reliable fallback for non-HTMX submit paths and richer content for visual checks.
- Fix:
  - Confirmed POST fallback forms for enroll/mark complete.
  - Progress now counts only published lessons with `event_type="done"`.
  - Expanded `seed_demo` with richer courses/modules/lessons/news/achievements/reviews and mixed lesson blocks (text/code/image/video/file).
- Files:
  - `src/apps/progress/services.py`
  - `src/apps/courses/views.py`
  - `src/templates/courses/partials/enroll_button.html`
  - `src/templates/courses/lesson_detail.html`
  - `src/apps/core/management/commands/seed_demo.py`

### 7) Review/auth form UX
- Problem: Review and auth forms looked technical and inconsistent.
- Fix:
  - Reworked review fields rendering with explicit labels/errors.
  - Updated login/register templates to render fields consistently (no raw `as_p`).
  - Added focused form-field styling, inline error style, and help-text list styling.
- Files:
  - `src/templates/reviews/partials/review_form_fields.html`
  - `src/templates/reviews/review_list.html`
  - `src/templates/core/home.html`
  - `src/templates/accounts/login.html`
